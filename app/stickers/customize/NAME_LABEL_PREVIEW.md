# 네임스티커 미리보기 학습 및 구현 설명

## 1. 고객 텍스트가 미리보기에 반영되는 방법

### 현재 동작 흐름

1. **고객 입력**  
   고객이 "Sticker Text" 입력란에 이름(또는 문구)을 입력합니다. (`customText` 상태)

2. **실시간 미리보기**  
   - 상품 이미지 위에 **텍스트 오버레이**가 같은 내용으로 표시됩니다.  
   - `customText`가 바뀔 때마다 미리보기 영역의 텍스트가 즉시 갱신됩니다.  
   - 텍스트 위치는 `textPosition`(기본 50%, 50%)으로 스티커 **안쪽 중앙**에 오도록 설정할 수 있습니다.

3. **캔버스 생성(장바구니/결제용)**  
   - `generateCustomizedImage()`에서 상품 이미지를 캔버스에 그린 뒤,  
     같은 `customText`를 `textPosition` 위치에 그려서 최종 이미지를 만듭니다.  
   - 이 이미지가 장바구니·결제 시 사용됩니다.

정리하면, **고객이 입력한 텍스트는 `customText` → 미리보기 오버레이 + 캔버스 그리기** 두 경로로 사용되며, 미리보기에서는 입력 즉시 스티커 안에 이름이 반영되어 확인할 수 있습니다.

---

## 2. Extra Large 네임스티커 치수 (45mm × 21mm)

### 시트 규격

| 항목 | 값 |
|------|-----|
| **스티커 1장 크기** | 45mm × 21mm (가로 × 세로) |
| **배치** | 가로 2줄 × 세로 6줄 (2 columns × 6 rows) |
| **스티커 간격** | 2mm |
| **시트당 스티커 수** | 12장 |

### 시트 전체 크기 (계산)

- **가로** `sheetWidthMm = cols × widthMm + (cols - 1) × gapMm`  
  → `2 × 45 + 1 × 2 = 92mm`
- **세로** `sheetHeightMm = rows × heightMm + (rows - 1) × gapMm`  
  → `6 × 21 + 5 × 2 = 136mm`

미리보기에서는 **시트 전체(92×136mm)** 비율의 그리드를 그리고, **각 스티커 칸(2×6 = 12칸)** 안에 고객이 입력한 이름이 들어가도록 표시합니다.

---

## 3. 미리보기에서 "스티커 안에 이름이 보이게" 하는 방법

### 구현 포인트

1. **스티커 1장 비율로 미리보기 영역 고정**  
   - Extra Large면 **45 : 21** 비율의 컨테이너를 사용합니다.  
   - CSS `aspectRatio: widthMm / heightMm` 로 한 장 스티커 모양을 유지합니다.

2. **텍스트를 스티커 안쪽에 표시**  
   - 텍스트 오버레이의 `left`, `top`을 `textPosition`(퍼센트)으로 두고,  
     기본값을 **(50, 50)**으로 해서 스티커 **정중앙**에 이름이 오도록 했습니다.  
   - `maxWidth: 85%` 로 긴 이름도 한 장 안에 들어가 보이도록 했습니다.

3. **사이즈별 치수 반영**  
   - `NAME_LABEL_SPEC` 에서 Extra Large(45×21), Large, Medium, Small, Round 치수를 정의했습니다.  
   - 선택된 상품의 `size`에 따라 해당 스펙의 비율이 미리보기에 적용됩니다.

### 코드 위치

- **치수 정의**: `app/stickers/customize/page.tsx` 내 `NAME_LABEL_SPEC`, `stickerSpec`, `stickerAspectRatio`
- **미리보기 컨테이너**: "Preview Canvas" 영역의 `aspectRatio` 적용된 `div`
- **텍스트 오버레이**: 같은 영역 안의 `customText` 렌더링 부분

## 4. 미리보기가 실제 시트보다 크게 보였던 이유 (15인치 화면 기준 학습)

### 원인

- 미리보기 크기는 **1mm당 몇 px로 그릴지**(`PX_PER_MM`)로 결정됩니다.
- **웹/CSS 기준 "실제 크기"**: 96 DPI 가정 → 1 inch = 96px → **1mm = 96/25.4 ≈ 3.78px**.
- 이전에는 **PX_PER_MM = 5** 를 사용해서:
  - 96mm 시트 가로 → 480px, 138mm 세로 → 690px 로 그렸고
  - 표준 비율(3.78px/mm)보다 **5/3.78 ≈ 1.32배** 커 보였습니다.

### 15인치 화면 기준

- 15인치 1920×1080: 화면 가로 약 **332mm**, 96mm 시트면 가로의 약 **29%** 가 실제 크기.
- 96 DPI 기준 "실제 96mm"는 **96 × 3.78 ≈ 363px**.
- PX_PER_MM=5일 때는 480px로 그려져, 같은 해상도에서도 **약 32% 더 크게** 보였습니다.

### 수정

- **PX_PER_MM = 96 / 25.4** 로 변경해, 1mm당 픽셀을 웹 표준에 맞춤.
- 이제 시트는 96mm → 약 363px, 138mm → 약 522px 로 그려져, 96 DPI 기준 모니터에서 **실제 치수에 가깝게** 보입니다.
- 15인치에서 브라우저 확대 100%, OS 배율 100%이면 시트가 손에 든 실제 크기와 비슷하게 느껴집니다.

---

## 5. 요약

- 고객이 입력한 텍스트는 **미리보기**와 **캔버스 생성** 모두에서 같은 `customText`로 반영됩니다.  
- Extra Large는 **45mm×21mm, 세로 6줄·가로 2줄, 간격 2mm**를 기준으로 하고,  
  미리보기는 **실제 시트 크기(96×138mm)**에 맞게 1mm = 96/25.4 px 로 그려, 15인치 등 일반 모니터에서 실제 크기에 가깝게 보이도록 했습니다.

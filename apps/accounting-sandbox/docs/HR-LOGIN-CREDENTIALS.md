# HR & Payroll - Login Credentials 가이드

## 📋 Login Credentials란?

**Login Credentials**는 직원이 SELPIC A 시스템에 로그인하여 본인의 타임시트를 작성하고 급여 정보를 확인할 수 있도록 하는 인증 정보입니다.

### 구성 요소
- **Employee ID**: 직원 고유 식별자 (필수)
- **Password**: 로그인 비밀번호 (선택사항)

---

## 🔐 직원 등록 시 Login Credentials 처리 방법

### ✅ **방법 1: 직원 등록 시 비밀번호 함께 설정 (권장)**

직원을 등록할 때 "Add New Employee" 폼의 **"Login Credentials"** 섹션에서 비밀번호를 입력할 수 있습니다.

**장점:**
- 직원 등록과 동시에 로그인 정보 설정
- 관리자가 초기 비밀번호를 설정하여 직원에게 전달 가능
- 즉시 시스템 사용 가능

**단점:**
- 관리자가 비밀번호를 알게 됨 (보안상 권장하지 않음)

**사용 시나리오:**
1. 관리자가 직원 등록 시 임시 비밀번호 설정 (예: `Temp123!`)
2. 직원에게 Employee ID와 임시 비밀번호 전달
3. 직원이 최초 로그인 후 비밀번호 변경 권장

---

### ✅ **방법 2: 직원 등록 시 비밀번호 비워두기 (더 안전)**

직원 등록 시 "Password" 필드를 비워두고, 나중에 직원이 직접 설정하도록 할 수 있습니다.

**현재 시스템 제한:**
- 현재는 비밀번호가 없으면 로그인할 수 없음
- 향후 "비밀번호 초기 설정" 기능 추가 필요

**개선 방안:**
- 직원이 최초 로그인 시 비밀번호 설정 페이지로 이동
- 또는 관리자가 나중에 비밀번호를 설정할 수 있는 기능 추가

---

## 🔄 현재 시스템 동작 방식

### 1. 직원 등록 시 (`EmployeeAddForm`)

```typescript
// Login Credentials 섹션
- Password (for employee login): 선택사항
- 비워두면: "Leave empty if not needed" 표시
- 입력하면: 해시하여 저장
```

### 2. 직원 로그인 시 (`EmployeeLogin`)

```typescript
// 로그인 프로세스
1. Employee ID 입력
2. Password 입력
3. IndexedDB에서 직원 정보 조회
4. 비밀번호 해시 비교
5. 일치하면 세션 생성 (24시간 유효)
```

### 3. 비밀번호 저장 방식

```typescript
// employee-auth.ts
- 평문 비밀번호 → 해시 변환
- 해시된 비밀번호만 IndexedDB에 저장
- 로그인 시 입력한 비밀번호를 해시하여 비교
```

## 📝 사용 가이드

### **시나리오 A: 관리자가 초기 비밀번호 설정**

1. **직원 등록**
   - HR & Payroll → Add Employee
   - 모든 필수 정보 입력
   - Login Credentials 섹션에서 비밀번호 입력 (예: `Welcome2024!`)
   - Save Employee 클릭

2. **직원에게 전달**
   - Employee ID: `EMP001`
   - 초기 비밀번호: `Welcome2024!`
   - 최초 로그인 후 비밀번호 변경 권장

3. **직원 로그인**
   - HR & Payroll 탭에서 Employee Login
   - Employee ID와 비밀번호 입력
   - 로그인 성공 → 타임시트 작성 가능

---

### **시나리오 B: 비밀번호 나중에 설정 (현재 제한)**

1. **직원 등록**
   - Login Credentials 섹션에서 비밀번호 비워두기
   - Save Employee 클릭

2. **문제점**
   - 현재 시스템에서는 비밀번호가 없으면 로그인 불가
   - `employee-auth.ts`의 `loginEmployee` 함수에서:
     ```typescript
     if (!employee.password) {
       return null // 로그인 실패
     }
     ```

3. **해결 방법 (향후 개선)**
   - 비밀번호가 없는 직원은 "초기 설정" 플로우로 이동
   - 또는 관리자가 나중에 비밀번호 설정 기능 추가

---

## ⚠️ 보안 고려사항

### 현재 구현
- ✅ 비밀번호 해시 저장 (평문 저장 안 함)
- ⚠️ 간단한 해시 함수 사용 (프로덕션에서는 bcrypt 권장)
- ✅ 세션 24시간 자동 만료
- ✅ 비활성 직원 로그인 차단

### 개선 권장사항
1. **bcrypt 사용**: 더 안전한 비밀번호 해시
2. **비밀번호 정책**: 최소 길이, 복잡도 요구사항
3. **비밀번호 변경 기능**: 직원이 직접 변경 가능
4. **비밀번호 재설정**: 분실 시 재설정 기능

## 🎯 결론

### **직원 등록 시 Login Credentials는 어떻게 해야 하나요?**

**권장 방법:**
1. ✅ **직원 등록 시 비밀번호를 함께 설정** (초기 비밀번호)
2. ✅ 직원에게 Employee ID와 초기 비밀번호 전달
3. ✅ 최초 로그인 후 비밀번호 변경 권장

**또는:**
1. ⚠️ 비밀번호를 비워두고 등록 (현재는 로그인 불가)
2. ⚠️ 나중에 관리자가 비밀번호 설정 (현재 기능 없음)

**결론: 현재는 직원 등록 시 비밀번호를 함께 설정하는 것이 가장 실용적입니다.**

# Stamps Hero Background Slides – 이미지 윗부분/아래부분이 잘리는 이유

## 현상
Stamps(및 Stickers, Phone Cases, Hot Goods) Hero Background Slides에서 업로드한 이미지의 **윗부분과 아래 부분이 보이지 않고**, 가운데 부분만 보이는 것처럼 보일 수 있습니다.

---

## 원인

### 1. 히어로 영역 비율
카테고리 히어로 섹션은 **가로는 화면 전체, 세로는 짧은** 형태입니다.

- **높이**: `min-h-[260px]` (모바일) ~ `min-h-[340px]` (데스크톱)  
  → 약 260px~340px
- **가로**: 100% (전체 너비)

즉, **가로로 넓고 세로로 짧은** 직사각형입니다.  
예: 1200×300 비율에 가까운 **가로형 영역**입니다.

### 2. 이미지 표시 방식: `background-size: cover`
슬라이드 이미지는 CSS **`background-size: cover`**(Tailwind `bg-cover`)로 표시됩니다.

- **cover** 동작:
  - 배경 이미지를 **비율 유지**한 채로 확대/축소해서,
  - **영역 전체를 꽉 채우도록** 맞춥니다.
  - 그 결과, **영역 밖으로 나가는 부분은 잘립니다**.

### 3. 세로가 긴 이미지를 넣으면
업로드한 이미지가 **세로가 더 긴 이미지**(세로형, 정사각형에 가까운 비율)인 경우:

- 영역을 **가로에 맞춰** 채우려면 이미지를 크게 키워야 하고,
- 그렇게 키우면 **위·아래가 영역 밖으로 나가서 잘립니다**.
- 그래서 **윗부분과 아래 부분이 보이지 않고**, 가운데 부분만 보이게 됩니다.

반대로 이미지가 **가로로 매우 넓은 이미지**라면, 위·아래는 다 보이고 **왼쪽/오른쪽**이 잘릴 수 있습니다.

---

## 정리

| 항목 | 내용 |
|------|------|
| **원인** | 히어로 영역이 “가로 넓고 세로 짧음” + `bg-cover` 로 꽉 채우기 |
| **결과** | 세로가 긴 이미지는 위·아래가 잘림, 가로가 긴 이미지는 좌·우가 잘림 |
| **“윗부분/아래 부분이 안 보인다”** | 업로드 이미지 비율이 히어로보다 **세로로 더 긴 경우**에 해당 |

---

## 관리자 측 대응 방법

1. **이미지 비율 맞추기**  
   히어로가 **가로형**이므로, **가로로 넓은 이미지**(예: 16:9, 3:1 등)를 사용하면 위·아래가 잘리지 않고 전체가 잘 맞습니다.

2. **코드 쪽 옵션 (구현 시)**  
   - **전체 이미지가 보이게**: `background-size: contain` 사용 → 위·아래까지 다 보이지만, 빈 여백(레터박스)이 생길 수 있음.  
   - **슬라이드별로 선택**: 슬라이드마다 “꽉 채우기(cover)” / “전체 보기(contain)”를 선택할 수 있게 하면, 이미지에 따라 관리자가 선택할 수 있습니다.

이 문서는 위와 같은 이유로 “Stamps Hero Background Slides에서 업로드 이미지의 윗부분과 아래 부분이 보이지 않는” 현상을 설명합니다.

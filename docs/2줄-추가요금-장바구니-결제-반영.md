# 2줄 옵션 추가 요금이 장바구니·결제에 반영되지 않을 때

## 1. 정상 동작 흐름 (학습)

1. **커스텀 페이지**  
   - 고객이 2줄 옵션 선택 시 `customizations`에 다음이 들어갑니다.  
     - `twoLineOption`: 표시용 문자열 (예: `"2 lines (Name & Phone) +$2.00"`)  
     - `twoLineSurchargeAmount`: 숫자만 문자열로 (예: `"2"`)  
   - 이 상태로 `addToCart(cartItem)` 호출.

2. **스토어 (lib/store.ts)**  
   - `addToCart`에서 `sanitizeCustomizations(item.customizations)` 호출.  
   - `sanitizeCustomizations`는 **`customizedimage`만 제거**하고, `twoLineOption`·`twoLineSurchargeAmount`는 그대로 유지.

3. **장바구니·결제 페이지**  
   - `getSurchargePerUnit(item.customizations)`로 단위당 추가 요금 계산.  
   - 계산된 금액을 `calculateSubtotal` 등에서 사용해 소계·총액에 반영.

## 2. 반영이 안 될 수 있는 이유

- **이전에 담긴 장바구니 데이터**  
  - 예전 코드로 담은 항목에는 `twoLineSurchargeAmount`가 없을 수 있음.  
  - 이 경우 `twoLineOption` 문자열만 있으면, **문자열에서 금액을 파싱**해서 쓰도록 보완해 두었음.

- **상품의 size 값**  
  - 2줄 지원 여부는 `displayProduct?.size`가  
    `'Large'`, `'Extra Large'`, `'Medium'`, `'대형'`, `'특대형'`, `'중형'`  
    일 때만 true.  
  - 관리자 입력이 `"large"`, `"Large "` 등이면 조건이 false가 되어, 커스텀 페이지에서 `twoLineOption`/`twoLineSurchargeAmount`가 아예 설정되지 않음.  
  - **해결**: 상품 등록 시 size를 위 값 중 하나로 정확히 입력.

- **localStorage/캐시**  
  - 오래된 장바구니가 남아 있으면, 예전 구조의 customizations만 있을 수 있음.  
  - **해결**: 장바구니 비우기 후, 2줄 선택 → 담기 → 장바구니/결제에서 다시 확인.

## 3. 수정 내용 (반영 방법)

- **장바구니·결제 공통**  
  - `getSurchargePerUnit`을 다음 순서로 동작하도록 변경했습니다.  
    1. `customizations.twoLineSurchargeAmount`가 있으면 그 값 사용.  
    2. 없으면 `customizations.twoLineOption` 문자열에서 `+$2.00` 형태를 찾아 파싱.  
    3. 그래도 없으면 기존처럼 다른 키 값들에서 `+$X.XX` 패턴 파싱.  
  - 이렇게 해서 **새로 담은 항목**(twoLineSurchargeAmount 있음)과 **예전에 담은 항목**(twoLineOption만 있음) 모두에서 2줄 추가 요금이 장바구니·결제에 적용되도록 했습니다.

- **커스텀 페이지**  
   - 2줄 선택 시 `twoLineOption`과 `twoLineSurchargeAmount`를 둘 다 넣는 기존 로직은 그대로 두었습니다.

## 4. 확인 방법

1. 2줄 지원 상품(Large/Medium/Extra Large 등) 선택.  
2. **Two lines** 선택 후 텍스트 입력.  
3. **Add to Cart** → 장바구니에서 해당 품목이  
   `$X.XX + $2.00 surcharge each` 형태로 보이고, Subtotal에 2불이 더해지는지 확인.  
4. 결제 페이지(Checkout)에서도 Subtotal·Total에 동일하게 반영되는지 확인.

이후에도 반영이 안 되면, 상품의 `size` 값과 장바구니에 저장된 `customizations`(개발자 도구 → Application → Local Storage → selpic-store)를 확인하면 됩니다.
